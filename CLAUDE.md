## Commands

```bash
# Development
pnpm dev          # Start dev server on port 3561 (webpack mode)
pnpm build        # Production build
pnpm lint         # ESLint via Next.js

# Content processing (run after adding images or new posts/books)
pnpm process      # Runs both blurhash and metadata generators
pnpm blurhash     # Generate blurhash/dimensions for images on R2 CDN
pnpm metadata     # Generate post-count.ts, post-slugs.ts, post-word-counts.json, and book equivalents

# Bundle analysis
pnpm analyze      # Build with bundle analyzer enabled
```

No test suite is configured.

## Architecture

This is a personal blog/portfolio site built with **Next.js 16 App Router**, React 19, TypeScript, Tailwind CSS, and MDX.

### Content System

Blog posts live in `src/posts/` as `.mdx` files. Each MDX file exports metadata at the top:

```ts
export const title = 'Title';
export const slug = 'slug';
export const tags = ['javascript'];
export const description = 'Description';
export const cover = 'cover';            // filename on R2 CDN
export const published = new Date(...);
export const shortCodes = ['code'];      // short URL aliases
export const bskyThreadId = '';          // Bluesky thread for comments
```

**Every new post must be manually added to `src/posts/index.ts`** (import + add to the `posts` object). The same pattern applies to books in `src/books/`.

After adding posts, run `pnpm metadata` to regenerate `src/post-count.ts`, `src/post-slugs.ts`, and `src/post-word-counts.json`.

A `src/posts/template.mdx` exists as a starting point.

### Routing

- `/blog/[slug]` — full-page blog post view
- `/@postModal/(...)blog/[slug]` — intercepting route that renders posts in a modal when navigated from the blog listing (parallel route `@postModal` in root layout)
- `/[shortCode]` — redirects short codes to the corresponding blog post slug
- `/library/[slug]` — book review pages (same MDX pattern as posts)
- `/api/og/*` — OG image generation routes using `@vercel/og`
- `/api/view` — tracks page views (PlanetScale DB)

### Infrastructure / Data

- **Images**: Served from Cloudflare R2 at `https://static.donley.xyz` (books: `https://library.static.donley.xyz`). Client is in `src/util/r2.ts`.
- **Image metadata**: Blurhash, dimensions, and blur placeholders are pre-generated by the Go tools and committed as JSON. Used for Next.js `Image` blur placeholders.
- **View counts**: PlanetScale MySQL DB accessed via `src/util/db.ts`. Tracked per-slug in `post_page_views` and aggregated in `post_page_views_aggregate`.
- **Edge Config** (`@vercel/edge-config`): Controls runtime settings like `blogPageSize`.
- **Redis** (`@upstash/redis`): Used for caching/rate limiting.

### Go Tooling (in subdirectories)

- `blurhash-generator/` — fetches images from R2, generates blurhash + dimensions, outputs JSON committed to repo
- `post-metadata-generator/` — scans `src/posts/` and `src/books/`, generates count/slug/word-count TS and JSON files
- `img-serve/` — local image proxy server for development

### Theming

Dark/light mode is stored in a cookie (`userTheme()` from `src/util/cookies.ts`). The `dark` class is applied to `<body>` server-side. Theme colors are defined in `src/theme/colors.ts`.

### Path Aliases

`@/` maps to `src/` (configured in tsconfig).
